## Key decisions made

- `add_enemy()` does array registration + `add_child()` + `enemy.died` connection in one call
- `receive_player_attack()` takes a target enemy and damage — target selection is game.gd's concern
- `run_enemy_turns()` lives on CombatEvent, not game.gd — keeps enemy count/state inside the event
- CombatEvent emits `player_attack_resolved(enemy, damage)` after applying damage, for UI/combat log
- `enemy.died` wired in `add_enemy()` (not `_on_setup()`) so it survives across phases
