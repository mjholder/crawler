## Key decisions made

- Enemies are Node2D (no physics needed for a room-based turn-based game)
- Signal-based separation: Player/Enemy emit signals; UI and game manager connect externally — see design.md — 2026-02-22
- Enemy uses extension hooks (_on_ready, _on_damaged, _on_death) so subclasses can customise behaviour without reimplementing core logic
- Player actions (attack, future: defend/item/flee) each emit turn_ended so the game manager stays in control of turn flow
